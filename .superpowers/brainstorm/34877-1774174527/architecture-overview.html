RAG-Datenfluss:
1. User stellt Frage (ggf. mit Pet-Kontext) → 2. FTS sucht relevante Wissens-Artikel in PostgreSQL → 3. Kontext + Pet-Daten + Frage werden als Prompt an Ollama gesendet → 4. Antwort wird per SSE gestreamt → 5. Chat-Nachricht wird bei Pet-Chats persistent gespeichert

Kern-Entscheidungen

RAG ohne Vektoren
PostgreSQL Full-Text-Search statt pgvector. Keine Extension, keine extra Services.
Ollama extern
Läuft auf dem Mac, nicht im Cluster. URL wird in AI-Settings konfiguriert, in DB gespeichert.
Chat pro Pet persistent
Pet-bezogene Gespräche in DB gespeichert. Allgemeine Chats nur session-basiert.
Spring AI ChatClient
Spring AI 2.0.0-M3 für Ollama-Kommunikation. Streaming via SSE.
