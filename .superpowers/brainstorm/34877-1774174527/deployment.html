Deployment — CF Context Handling

Safe org/space switching in deploy-cf.sh

deploy-cf.sh — Context-Safe Deployment Flow
#!/usr/bin/env bash
# deploy-cf.sh — deploys petclinic to demo/spring, restores previous context
# 1. Save current CF context
PREV_ORG=$(cf target | grep "org:" | awk '{print $2}')
PREV_SPACE=$(cf target | grep "space:" | awk '{print $2}')
# 2. Switch to demo/spring
cf target -o demo -s spring
# 3. Create service if not exists
cf create-service postgresql default petclinic-db
# 4. Build & push
./gradlew bootJar
cf push
# 5. Restore previous context
cf target -o "$PREV_ORG" -s "$PREV_SPACE"
echo "Restored context: $PREV_ORG/$PREV_SPACE"
Trap handler: Context is also restored on script failure (trap ... EXIT)
So even if cf push fails, you're back on kappman/app.
