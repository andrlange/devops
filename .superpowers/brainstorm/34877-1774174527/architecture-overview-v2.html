RAG + WebSocket Streaming Fluss:
1. User sendet Nachricht via STOMP (/app/chat.send) → 2. FTS sucht relevante Wissens-Artikel in PostgreSQL → 3. Kontext + Pet-Daten + Frage als Prompt an Ollama → 4. Ollama streamt Tokens → Spring AI Flux → STOMP push an /topic/chat.{sessionId} → 5. Browser rendert Token-für-Token → 6. Bei Pet-Chats: vollständige Antwort in DB persistiert

WebSocket vs SSE

WebSocket (STOMP + SockJS)

  • Bidirektional — User kann abbrechen/neue Frage senden
  • Stabil durch Traefik/Reverse-Proxies
  • SockJS Fallback (long-polling) wenn WS blockiert
  • Connection-State im Client sichtbar
  • Spring Boot erstklassiger Support

SSE (verworfen)

  • Unidirektional — kein Abbruch möglich
  • Proxy-Timeout-Probleme (Traefik, nginx)
  • Reconnect-Logik muss selbst gebaut werden
  • Browser-Limit: max 6 SSE-Verbindungen pro Domain
